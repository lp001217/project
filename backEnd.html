Overall architecture description

1) Technology stack

  • Server: Node.js + Express
  • Database: MongoDB (modeled using Mongoose)
  • Authentication: JWT (used for user login status verification, protection of "My Page" "Shopping Cart" and other interfaces)
  • Payment: Integrate Alipay / WeChat / PayPal official SDK (handle payment callback and order status synchronization)

2) Core module

  • User module: manage user accounts, delivery addresses, etc. (corresponding to "My Page").
  • Product module: supports product classification, search, and details display (corresponding to "Cultural and Creative Mall" "Product Details Page").
  • Transaction module: shopping cart, order, payment process (corresponding to "Shopping Cart" "Payment Page" "Order Management Page").
  • Competition module: competition registration, work submission, voting (corresponding to "Competition Homepage" "Work Submission Page" "Voting Page").

Core function implementation description

1) Product classification and search optimization

  • Category screening: Direct filtering through the database category and sellerType fields, high performance.
  • Search function: Using MongoDB's text index (text index), support keyword fuzzy search of product titles and descriptions to improve search speed.

2) Shopping cart and user association

  • Shopping cart data is stored in the cart array of the User model. JWT authentication is used to ensure that users can only operate their own shopping carts to avoid unauthorized access.

3) Payment process and order state machine

  • The payment interface must strictly follow the security specifications of the third-party payment platform (such as signature verification, asynchronous callback processing) to ensure that the order status and payment results are ultimately consistent (using database transactions or status retry mechanisms).

4) Anti-brushing of competition voting

  • By creating a unique composite index { userId: 1, workId: 1, competitionId: 1 } in the Vote collection, ensure that each user can only vote once for each work in each competition.
